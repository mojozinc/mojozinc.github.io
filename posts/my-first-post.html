First Post: Setting Up a Frictionless Writing Platform

June 29, 2026 • 2 min read

For years, I had a placeholder portfolio page that gathered digital dust. I wanted to write more, but the friction of maintaining a complex blogging setup—managing static site generator upgrades, configuring node modules, or tweaking Tailwind configuration files—frequently stopped me before I even started.

This is a new start. I have redesigned this site to be as close to a 2000s programmer-essayist style page as possible: simple, text-centric, fast, and completely free of build steps.

How this platform works

Writing a new post is as simple as:

  1. Copying this HTML file (e.g., to posts/my-new-essay.html).
  2. Updating the title, meta tags, and writing standard HTML elements like headers, lists, and paragraphs.
  3. Adding the post link to the list in index.html.
  4. Running git commit and pushing to GitHub.
"The best writing setup is the one that stays completely out of your way."

Rich Text & Code Formatting

Because this is a developer site, clean code formatting is a first-class citizen. Here is an example of inline code like const x = 42, and a standard pre-formatted block below: 

// A simple, pure CSS/JS theme toggler
const themeToggleBtn = document.getElementById('theme-toggle');
themeToggleBtn.addEventListener('click', () => {
    const isDark = document.documentElement.getAttribute('data-theme') === 'dark';
    document.documentElement.setAttribute('data-theme', isDark ? 'light' : 'dark');
});

By keeping the HTML clean and leveraging semantic tags, the reader's attention remains exactly where it belongs: on the ideas.
